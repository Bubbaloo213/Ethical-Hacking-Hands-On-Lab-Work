# Basic HTTP GET/Response Interaction 

The objective of this exercise is to examine how the HTTP protocol works by analyzing network traffic with Wireshark. You will use cURL to generate controlled HTTP requests—specifically forcing IPv4 and unencrypted (plain) HTTP—while capturing the packets in Wireshark for detailed inspection. The focus of the analysis is the HTTP GET request. A GET request asks the server for a specific resource, and the server responds with a 200 OK message that includes both response headers and the message body.

## Capture, filter, and understand HTTP traffic in Wireshark

### Start Wireshark Capture

1) Start Wireshark (don't start capture yet)
2) Enter display filter: **http**
3) Select **Capture** > **Options** > Select on the active interface (Ethernet0) > **Start** Wireshark packet capture

### Make HTTP Request

4) Open **Command Prompt** and execute this CURL command:

  ```javascript
  curl --http1.1 -4 -o HTTP-wireshark-file1.html -H "Accept-Language: en-US,en;q=0.9" http://gaia.cs.umass.edu/wireshark-labs/HTTP-wireshark-file1.html
  ```

### Analyze in Wireshark
6) You should see two HTTP messages:
      * GET request (from your machine to server)
      * HTTP OK response (from server to you)

7) Click on the HTTP Get Message and answer the following questions:
   * Based on your inspection in Wireshark, are we running HTTP version 1.0 or 1.1? **ANSWER:** 1.1
   * Based on your Wireshark inspection, what language does your host indicate that it can accept to the server?  List one of the values listed in the Accept-Language field. **ANSWER:** en-us
   * What is the IP address of your computer? **ANSWER:** 192.168.238.131

<details>
  <summary> Why 200? </summary>
              The status code 200 in the HTTP protocol indicates that the request made by the client (such as a web browser) was successful. When a server returns a 200 status code, it means that the server has processed the request and is returning the requested resource (like a web page, image, or data) without any errors.
