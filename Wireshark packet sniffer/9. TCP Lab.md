# TCP Lab

The objective of this exercise is to review the behavior of the TCP protocol in detail by analyzing a trace of TCP segments sent and received in transferring a 150KB file (containing the text of Lewis Carroll's Alice's Adventures in Wonderland) from your computer to a remote server.

To begin, I will use Wireshark to capture the packet trace of this TCP file transfer. This will be done by accessing a web page that allows you to select a file stored locally on your computer (the ASCII text of Alice in Wonderland) and then upload it to a web server using the HTTP POST method.

<details>
  <summary> ** Why HTTP POST instead of GET method? ** </summary>
  The POST method is used instead of GET because our goal is to transfer a substantial amount of data from your computer to the server. While GET requests are typically used to retrieve data, POST requests are designed to send data.
</details>

## Download the Alice in Wonderland Text File

**Note:** These next steps are to be executed in the Windows machine

1. Open Command Prompt and execute:

```javascript
  curl -4 --http1.1 -o alice.txt http://gaia.cs.umass.edu/wireshark-labs/alice.txt
  ```

<details>
  <summary> ** Understanding the curl command: ** </summary>
  <br />
  - -4: Forces IPv4 only (prevents IPv6 which can complicate analysis)
  <br />
  - --http1.1: Uses HTTP/1.1 without encryption (no TLS/HTTPS)
  <br />
  - -o alice.txt: Saves output to file named alice.txt
</details>

## Upload the File using curl

7. Within command prompt, execute the following command so your computer uploads the Alice in Wonderland form (**Alice.txt**) to a web server using the HTTP POST method.

```javascript
  curl -4 --http1.1 -X POST -F "file=@alice.txt" http://gaia.cs.umass.edu/wireshark-labs/lab3-1-reply.htm
  ```

  <img width="1133" height="247" alt="image" src="https://github.com/user-attachments/assets/003a23e3-6917-42db-8b01-88e0396145e1" />


<details>
  <summary> ** Understanding the curl command: ** </summary>
    <br />
  - -4: Forces IPv4 only (prevents IPv6 complications)
    <br />
  - --http1.1: Uses HTTP/1.1 without TLS encryption
    <br />
  - -X POST: Specifies HTTP POST method
    <br />
  - -F "file=@alice.txt": Sends file as multipart/form-data with field name "file"
    <br />
  - URL: Goes directly to the form action endpoint
</details>

  * This creates a single, clean TCP connection without browser overhead, making your packet analysis much clearer! You should see HTML response indicating successful upload.
