## Answer the following questions based on the contents of the Ethernet frame containing the HTTP GET message
1. What is the 48-bit Ethernet address of your computer? **ANSWER:** 00:0c:29:82:b9:c2
2. Locate the 48-bit destination address in the Ethernet frame. What device has this as its Ethernet address? **ANSWER:** My local router
3. Provide the hexadecimal value (in the 0x0000 format) for the two-byte Frame type field.  **ANSWER:** 0x0800
4. What upper layer protocol does this correspond to? **ANSWER:** IPv4

## Answer the following questions based on ARP messages.
1. What is the hexadecimal value for the destination address in the Ethernet frame containing the ARP request message? **ANSWER:** ff:ff:ff:ff:ff
2. Provide the hexadecimal value (in the 0x0000 format) for the two-byte Frame type field (in Ethernet section). **ANSWER:** 0x0806
3. What upper layer protocol does this correspond to? **ANSWER:** ARP

   <img width="1398" height="698" alt="image" src="https://github.com/user-attachments/assets/92e083da-54d0-4dc7-aa9a-d99cfd8e567a" />

4. What is the value of the request opcode field within the ARP-payload part of the Ethernet frame when an ARP request is made? **ANSWER:** 1
5. Does the ARP request message contain the IP address of the sender? **ANSWER:** Yes
6. Where in the ARP request does the “question” appear – the Ethernet address of the machine whose corresponding IP address is being queried? **ANSWER:** The “Target MAC address” is 00:00:00:00:00:00 to question the machine whose IP address is being queried.
