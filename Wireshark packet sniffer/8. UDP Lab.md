# UDP Lab

The objective of this exercise is to generate, capture, and analyze UDP Traffic. DNS is the most common UDP protocol you'll encounter.

## Generating and Capturing UDP Traffic

** Note: These next steps are to be executed in the Windows machine

1. Open Wireshark and start capturing packets
2. Open Command Line and run ***nslookup*** commands:
  
  Basic DNS query to Google's DNS server
```javascript
  nslookup www.google.com 8.8.8.8
  ```

  Query Cloudflare's DNS server
```javascript
  nslookup www.github.com 1.1.1.1
  ```

  Query your university website
```javascript
  nslookup www.nyu.edu 8.8.4.4
  ```

  Query for mail servers
```javascript
  nslookup -type=MX google.com 8.8.8.8
  ```

  Query for text records
```javascript
  nslookup -type=TXT google.com 8.8.8.8
  ```

<details>
  <summary> ** What's happening? ** </summary>
        * nslookup sends DNS queries using UDP port 53
        * 8.8.8.8 is Google's public DNS server
        * 1.1.1.1 is Cloudflare's public DNS server
        * Different query types generate different packet sizes
</details>

3. Check if the dns flush occurred correctly by executing:

  Windows PowerShell has Resolve-DnsName for more details
```javascript
  Resolve-DnsName www.google.com -Server 8.8.8.8 -Type A
  ```

  Query multiple record types
```javascript
  Resolve-DnsName google.com -Server 8.8.8.8 -Type MX
  ```

```javascript
  Resolve-DnsName google.com -Server 8.8.8.8 -Type TXT
  ```

 Clear DNS cache to generate fresh queries
```javascript
  ipconfig /flushdns
  ```

## Generate Additional UDP Traffic

4. Release current IP address
```javascript
  ipconfig /release
  ```

5. Wait 2 seconds, then renew
```javascript
  ipconfig /renew
  ```

6. Query time server (generates NTP traffic on UDP port 123)
```javascript
  w32tm /stripchart /computer:time.windows.com /samples:4
  ```
