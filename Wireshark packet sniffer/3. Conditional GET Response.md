# Conditional GET/Response to check if a Cached File has changed

The objective of this exercise is to understand how web browsers use caching by analyzing conditional HTTP GET requests. You will observe how browsers include freshness validators—such as If-Modified-Since (a timestamp) and If-None-Match with an ETag (a version identifier)—when requesting a resource that may already be cached. By comparing the first and second request/response exchanges, you will see how the server responds with 304 Not Modified (headers only, no body) when the resource has not changed, or 200 OK (headers plus body) when it has been updated. 

<details>
  <summary> Did You Know? </summary>
              Browsers use caching to save bandwidth by storing copies of web resources locally. When checking if a cached resource is still valid, they send a conditional GET request with freshness validators like If-Modified-Since (timestamp) or If-None-Match (ETag).

If the resource hasn't changed, the server responds with a 304 Not Modified status, sending only headers (no body), saving bandwidth.
If it has changed, the server responds with a 200 OK status and includes the updated resource in the body.
This mechanism allows browsers to efficiently manage cached content and reduce unnecessary data transfers. 

</details>

## First (1st) Web Browser (Client) request to visit a page
4) Start new Wireshark capture
5) Web browser (client) asks to access http://gaia.cs.umass.edu by running the following command:
   
```javascript
  curl --ipv4 -O --etag-save etag.txt http://gaia.cs.umass.edu/wireshark-labs/HTTP-wireshark-file2.html
  ```
  - Web server sends the page to the web browser (client) (200 OK)
  - Web browser (client) saves a copy (cache)
    
## Second (2nd) Web Browser (Client) request to visit the same page
5) Without stopping Wireshark capture, immediately run the below command on Windows Powershell:

```javascript
curl.exe --ipv4 --etag-compare etag.txt http://gaia.cs.umass.edu/wireshark-labs/HTTP-wireshark-file2.html
```
  - Instead of downloading the page again, the web browser (client) asks the web server conditionally to only send the page if it has changed (conditional GET).

  - Two possible responses from the web server
      - Case 1: The page has NOT changed (304 Not Modified) and the web browser (client) can use the copy it already has cached.
      - Case 2: The copy HAS changed (200 OK) so the web server sends the updated page. Web browser (client) updates its cache.


6) Stop Wireshark capture, save your capture.


<details>
  <summary> ** Note ** </summary>
             CURL doesn't cache by default, but we add an If-Modified-Since alternative (If-None-Match). Normally, your browser handles this for you! We use the etag flag in Windows. An ETag is a server-generated fingerprint for a file (a version ID). So if the file changes, the fingerprint changes too!

</details>
