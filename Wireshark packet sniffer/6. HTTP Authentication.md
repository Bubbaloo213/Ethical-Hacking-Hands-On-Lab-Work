# HTTP Authentication

When you try to access a protected webpage without logging in, the server responds with a "401 Unauthorized" message, indicating that you need to prove your identity. It includes a "WWW-Authenticate" header as a challenge. You then send your login credentials back to the server in an "Authorization: Basic " format, where Base64 is a method of encoding your information. However, Base64 is not secure, as it can be easily decoded by anyone intercepting the data. This is why using HTTPS, a secure version of HTTP that encrypts data, is essential to protect your login information from being exposed. Tools like Wireshark can capture network traffic, allowing you to see the Authorization header and decode it, highlighting the importance of using HTTPS for security.

## Request Without Credentials

1) Start Wireshark capture
2) Run:
```javascript
curl --http1.1 -4 -o HTTP-wireshark-file5.html http://gaia.cs.umass.edu/wireshark-labs/protected_pages/HTTP-wireshark-file5.html
  ```
<br />

## Request With Credentials

3) Without stopping capture, run:

```javascript
curl --http1.1 -4  -u "wireshark-students:network" -o HTTP-wireshark-file5.html http://gaia.cs.umass.edu/wireshark-labs/protected_pages/HTTP-wireshark-file5.html
  ```
<br />

4)  **Stop** capture after completion
