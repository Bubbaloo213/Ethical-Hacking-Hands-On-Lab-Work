# HTTP Authentication

When a user attempts to access a protected webpage without authentication, the server responds with a 401 Unauthorized status and a WWW‑Authenticate header, indicating that credentials are required. The browser then resends the request with an Authorization: Basic header containing the user’s credentials encoded in Base64. Because Base64 is only an encoding method and not encryption, these credentials can be easily decoded if intercepted. Network analysis tools like Wireshark can capture and decode this information, demonstrating why HTTPS encryption is essential to protect credentials from exposure during transmission.

## Request Without Credentials

1) Start Wireshark capture
2) Within Command Prompt, run the following command:
```javascript
curl --http1.1 -4 -o HTTP-wireshark-file5.html http://gaia.cs.umass.edu/wireshark-labs/protected_pages/HTTP-wireshark-file5.html
  ```
<br />

## Request With Credentials

3) Without stopping capture, run:

```javascript
curl --http1.1 -4  -u "wireshark-students:network" -o HTTP-wireshark-file5.html http://gaia.cs.umass.edu/wireshark-labs/protected_pages/HTTP-wireshark-file5.html
  ```
<br />

4)  **Stop** Wireshark capture after completion
