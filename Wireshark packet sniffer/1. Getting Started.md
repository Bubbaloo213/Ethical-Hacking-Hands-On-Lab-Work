## Understand how to capture packets using Wireshark

<details>
  <summary> What is Wireshark? </summary>
              Wireshark is a widely used open-source network protocol analyzer (aka "packet sniffer") that allows users to capture and interactively browse the traffic running on a computer network.  It supports a wide range of protocols, allowing users to dissect packet details and understand their interactions. 

</details>

    A hardware or software component that connects a device to a network, enabling communication with other devices. It serves as the point of interaction between the device and the network, allowing data to be sent and received. Network interfaces can be physical, such as Ethernet ports or Wi-Fi adapters, or virtual, such as those created by software for virtual machines or network simulations.

## Generate HTTP traffic for analysis in Wireshark using the CURL command-line tool 

The goal is to create network traffic that can be easily analyzed, specifically focusing on HTTP rather than HTTPS.

<details>
  <summary> What is CURL? </summary>
              CURL is a command-line tool that allows users to make network requests. It is preferred here because modern web browsers often use HTTPS (which encrypts data) and may use IPv6 (which can complicate analysis).
</details>

5) Open **Command Prompt** and execute this CURL command:

  ```javascript
  curl -4 --http1.1 http://gaia.cs.umass.edu/wireshark-labs/INTRO-wireshark-file1.html
  ```

* Analyzing HTTP traffic is easier when it is unencrypted. By using CURL, you can force the use of plain HTTP, making it simpler to capture and analyze the packets in Wireshark.

## Be capable of identifying different protocols in captured network traffic.

  <img width="1024" height="728" alt="image" src="https://github.com/user-attachments/assets/588203c7-6d23-4216-a2c4-df6b611474c9" />

* List of protocols and brief explanation

1) ARP (Address Resolution Protocol): This protocol helps devices on a local network find the physical address (MAC address) of another device based on its IP address.

2) DNS (Domain Name System): DNS translates human-friendly domain names (like www.example.com) into IP addresses that computers use to identify each other on the network.

3) TCP (Transmission Control Protocol): TCP is a protocol that ensures reliable communication between devices. It breaks data into packets, sends them, and makes sure they arrive correctly and in order.

4) HTTP (Hypertext Transfer Protocol): HTTP is the protocol used for transferring web pages on the internet. When you visit a website, your browser uses HTTP to request and receive the page content.

5) TLSv1.3 (Transport Layer Security version 1.3): This is a security protocol that encrypts data sent over the internet, ensuring that communications remain private and secure. It’s commonly used in conjunction with HTTP to create HTTPS, which is secure web browsing.

6) mDNS (Multicast Domain Name System): mDNS is a protocol used for resolving hostnames to IP addresses within small networks without a central DNS server. It’s often used in local networks for devices like printers and smart home devices.

7) NBNS (NetBIOS Name Service): NBNS is used for name resolution in Windows networks. It helps devices find each other by their NetBIOS names, which are often used in local area networks.

8) SSDP (Simple Service Discovery Protocol): SSDP is used for discovering devices and services on a local network.

In summary, these protocols help devices communicate, find each other, and ensure secure data transfer over networks. Each serves a specific purpose in the overall functioning of networked systems.
